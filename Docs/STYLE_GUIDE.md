# GoDoFramework 文档写作规范

本文定义用户文档的内容边界、推荐结构和自动检查规则。目标是让读者先完成任务，再按需理解设计，而不是把源码目录直接转换成网页目录。

## 基本原则

- 一页只承担一种主要职责：快速开始、教程、Recipe、模块指南、API Reference、架构说明或故障排查。
- 第一段直接说明读者能完成什么，以及这项能力不解决什么。
- 先给最小可运行路径，再解释高级选项、失败语义和实现背景。
- 代码示例使用项目中存在的 API；不确定的 API 必须先由源码和编译确认。
- 用户指南不复制完整 API 签名，详细成员信息链接到 API Reference。
- 编译、自动回归和 Godot 手动验证必须分别描述，不用“已验证”笼统替代。

## 页面类型

### 快速开始

目标是在最短路径内得到一个可观察结果。只保留必要前置条件、操作步骤、结果和下一步，不展开完整设计。

### 教程

从零完成一个完整目标。步骤必须有先后关系，并说明最终场景、代码或运行结果。

### Recipe

解决一个独立问题，例如新增 Procedure、UI View 或存档字段。一篇 Recipe 不同时介绍多个模块的完整设计。

### 模块指南

说明模块边界、适用场景、公共入口、失败语义、生命周期、性能和验证范围。对应模块的 `USAGE.md` 是详细行为说明的事实来源。

### API Reference

由 `GoDo.*` public API 的 XML 注释自动生成。XML 注释必须说明职责、参数、返回值、异常和重要生命周期约束，不能依赖只有源码作者知道的上下文。

### 架构与维护者文档

解释依赖方向和设计决策，不进入新用户的首要阅读路径。历史设想、路线和协作规则不混入用户指南。

### 故障排查

按“现象 → 可能原因 → 确认方式 → 解决方法”组织。错误文本和命令可以精确引用，结论必须区分已确认原因与推测。

## 推荐结构

模块指南使用 [MODULE_TEMPLATE.md](Templates/MODULE_TEMPLATE.md)；Recipe 使用 [RECIPE_TEMPLATE.md](Templates/RECIPE_TEMPLATE.md)。已有文档可以逐步迁移，不要求为满足标题名称而机械重写内容。

## 标题与导航

- 每个文件必须恰好包含一个 `# 一级标题`。
- 标题层级逐级下降，不能从 `##` 直接跳到 `####`。
- 标题表达用户概念或任务，不使用只有目录上下文才能理解的“其他”“更多”等名称。
- 文件移动后保持稳定的逻辑页面 ID；双语页面使用同一个输出路径。
- 页面标题可以翻译，C# 标识符、资源键和命名空间保持源码拼写。

## 代码与命令

- 围栏代码块必须标注语言，例如 `csharp`、`text`、`powershell`、`json`。
- 示例优先展示完整的最小调用边界，省略内容时明确使用注释或省略号。
- 代码中的 `Ui`、`Bgm`、`Sfx` 遵循项目 C# 命名；正文描述概念时使用 UI、BGM、SFX。
- 命令注明执行目录；会修改项目、场景或外部状态的命令必须说明影响。
- 不在文档中写入密钥、Token、真实服务地址或用户本机绝对路径。

## Godot 与框架约束

- 涉及 Godot 版本时明确写出适用版本；当前基线为 Godot 4.7 C# / .NET 8。
- 涉及输入、物理、节点生命周期、场景切换或编辑器配置时，列出 Godot 手动验证步骤。
- 高频路径说明额外分配、节点查找、LINQ、资源加载或字符串构造风险。
- 失败行为明确区分异常、返回值、正常分支、降级和日志上报。
- 生命周期说明初始化条件、主线程限制、订阅解绑和退出清理。
- 模块成熟度与平台状态必须和 `AI/ARCHITECTURE.md` 一致。

## 图片与可访问性

- 只有编辑器操作、节点树或视觉结果难以用短文本表达时才使用截图。
- 图片必须提供能表达关键信息的替代文本。
- 截图避免包含用户名、本机路径、密钥、项目私有内容和无关桌面区域。
- 界面变化后同步更新截图；无法确认版本时不把截图当作唯一操作依据。

## 双语规则

- 中文 `USAGE.md` 的英文翻译放在同目录的 `USAGE.en.md`。
- 中文 Recipe `Name.md` 的英文翻译命名为 `Name.en.md`。
- 独立站点页面放在 `Docs/i18n/zh-cn/` 与 `Docs/i18n/en-us/` 的对应路径。
- 翻译文件必须包含 `translation_of` 和 `translation_source_hash` front matter。
- 翻译以中文事实来源为准，但不逐字直译；英文应保持技术含义、失败语义和验证范围一致。
- public API 标识符不翻译。当前英文站的 API 页面复用中文 XML 注释，直到项目单独决定 API 注释语言策略。
- 缺少英文翻译时不生成机器翻译占位页；语言切换回到英文首页。

翻译 front matter 示例：

```yaml
---
translation_of: Docs/i18n/zh-cn/index.md
translation_source_hash: sha256:<中文源文件摘要>
---
```

中文源文件更新后，`python Docs/build_docs.py check` 会拒绝过期翻译。完成复核后更新摘要，不能只更新摘要而不检查正文。

## 自动检查

`python Docs/build_docs.py check` 当前检查：

- 一级标题数量；
- 标题层级；
- 围栏代码块语言与闭合；
- 重复输出路径；
- 翻译源文件与 SHA-256 摘要；
- DocFX 内部链接、锚点和 API 引用警告。
- 根入口、语言站点、搜索索引、sitemap、关键页面与语言切换目标等最终发布产物。

模块章节完整性作为提示输出，避免一次性强迫历史文档机械改名。新增模块应直接使用模板满足推荐结构。
