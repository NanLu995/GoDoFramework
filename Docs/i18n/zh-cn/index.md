# GoDoFramework

GoDoFramework 是面向 Godot 4.7 C# / .NET 的轻量游戏开发框架。它提供跨项目可复用的通信、诊断、资源、生命周期和运行时服务，让游戏项目把主要精力放在具体玩法与内容上。

## 适合谁

- 使用 Godot C# 制作中大型项目，希望统一场景、UI、音频、输入、存档和流程边界的开发者。
- 需要明确失败语义、生命周期、线程和验证范围的团队。
- 希望 AI 或自动化工具能够按稳定约定协作的项目。

框架不会替代 Godot，也不包含角色、战斗、关卡等具体玩法逻辑。所有框架 API 位于 `GoDo` 命名空间。

## 从这里开始

1. 阅读“5 分钟快速开始”，完成复制、启用和 Runtime 安装。
2. 阅读“使用框架制作游戏”和“推荐项目结构”，建立业务入口。
3. 按任务查找 Recipe，按模块查找失败语义和生命周期。
4. 需要精确成员签名时使用 API Reference。

## 当前状态

各模块成熟度和平台验证范围以“框架架构”为准。标记为首版完成或核心接入的模块仍可能调整，不能仅凭页面存在就视为稳定兼容性基线。
